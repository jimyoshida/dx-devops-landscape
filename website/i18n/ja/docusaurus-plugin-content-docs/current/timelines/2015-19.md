# タイムライン - 2015-19

## 2015

🏢 2013 年までに、Prometheus は SoundCloud での本番環境監視用に導入されました。公式な公開発表は 2015 年 1 月に行われました。

📊 Project Jupyter は、複数のプログラミング言語にわたるインタラクティブコンピューティングの為のオープンソースソフトウェア、オープン標準、およびサービスを開発するプロジェクトであり、2014 年に IPython から Fernando Pérez と Brian Granger によってスピンオフされ、2015 年 2 月に正式に確立されました。

🧠 Keras はオープンソースソフトウェアライブラリであり、人工神経網への Python インターフェースを提供し、TensorFlow ライブラリの為のインターフェースとして機能し、2015 年 3 月 27 日に初期リリースされました。

🖥️ 2015 年 4 月 8 日、GitHub は Git Large File Storage (Git LFS) を発表しました。これにより、ユーザーは Git 内の大きなバイナリファイルを保存および操作できます。

🖥️ Visual Studio Code は Microsoft が Windows、Linux、および macOS 用に開発したソースコードエディタであり、2015 年 4 月 29 日に Build カンファレンスで初めて発表され、その直後にプレビュービルドがリリースされました。

⚙️ 2015 年 4 月、Debian 8.0 (Jessie) は systemd をデフォルトの init システムとしてリリースされました。同じ月に、Ubuntu 15.04 も systemd に正式に切り替わってリリースされました。

🌐 Discord は 2015 年 5 月 13 日に公開リリースされた、ゲームコミュニティが一緒にプレイしながらコミュニケーションするために特別に設計されたプラットフォームです。

📜 最初の安定版リリースである Rust 1.0 は 2015 年 5 月 15 日に発表されました。

📜 第 6 版である ECMAScript 6 (ES6) 、後に ECMAScript 2015 に改名された、は 2015 年 6 月に確定されました。これはクラス宣言および ES6 モジュールを含む複雑なアプリケーション作成の為の重要な新しい構文を追加しました。ES6 は「アロー関数」構文をサポートしており、=> 記号は匿名関数のパラメータリストから本体を分離します。

☁️ Open Container Initiative (OCI) は 2015 年 6 月 22 日に DockerCon で Docker、CoreOS、および他の業界主導者によって Linux Foundation の下で起動され、オペレーティングシステムレベルの仮想化のためのオープン標準を設計する為のものです。
OCI は runc を開発しており、これは彼らの仕様を実装するコンテナランタイムであり、他の高レベルツールの基礎として機能します。runc は 2015 年 7 月にバージョン 0.0.1 として最初にリリースされました。

🏢 Kubernetes 1.0 は 2015 年 7 月 21 日にリリースされました。Google は Linux Foundation と協力して Cloud Native Computing Foundation (CNCF) を形成し、Kubernetes をシード技術として提供しました。

⚙️ Windows 10 は 2015 年 7 月 29 日にリリースされ、継続的な更新を伴う「Windows as a Service」モデルを導入し、Windows 8 のタイルベースの実験の後、スタートメニューを復元し、Microsoft Edge ブラウザーと Cortana 仮想アシスタントを統合しました。

📜 Go 1.5 は 2015 年 8 月 19 日にリリースされました。これはコンパイラーとランタイムツールチーン全体が C から Go に書き直された画期的なリリースであり、言語を自己ホスティング可能にしました。

🌐 Phoenix v1.0 は 2015 年 9 月 1 日にリリースされ、生産的で信頼性の高い Web フレームワークを提供し、Elixir エコシステムの基礎となりました。

🔐 FIDO2 Project は FIDO Alliance と World Wide Web Consortium (W3C) の共同努力であり、その目標は Web の為の強力な認証を作成することです。FIDO2 の中核は W3C Web Authentication (WebAuthn) 標準と FIDO Client to Authenticator Protocol 2 (CTAP2) で構成され、FIDO 2.0 Proposed Standard は 2015 年 9 月 4 日にリリースされました。

🏢 2015 年 9 月 14 日、Let's Encrypt は最初の証明書を発行しました。これは helloworld.letsencrypt.org ドメイン用のものでした。同じ日に、ISRG は Mozilla、Microsoft、Google、および Apple へのルートプログラムアプリケーションを提出しました。

🏢 2015 年 9 月に、Node.js v0.12 と io.js v3.3 は Node v4.0 に統合されました。

🌐 Vue.js はオープンソースモデル・ビュー・ビューモデル フロントエンド JavaScript フレームワークであり、ユーザーインターフェースおよびシングルページアプリケーション構築の為のもので、Evan You により作成されました。プロジェクトへの最初のソースコードコミットは 2013 年 7 月に遡り、Vue は 2014 年 2 月に最初にリリースされ、Vue.js 1.0 は 2015 年 10 月 27 日にリリースされました。

☁️ Serverless Framework は Node.js を使用して作成されたオープンソース Web フレームワークであり、AWS Lambda 上のアプリケーション構築の為に開発された最初のフレームワークです。Austen Collins により開発され、フルタイムチームにより保守されており、2015 年 10 月に初期リリースされました。

🧠 TensorFlow は機械学習および人工知能の為の無料のオープンソースソフトウェアライブラリであり、様々なタスクに使用可能ですが、特に深いニューラルネットワークのトレーニングと推論に焦点を当てており、2015 年 11 月 9 日に初期リリースされました。

🏢 2015 年 11 月 18 日、Visual Studio Code のソースは MIT ライセンスの下でリリースされ、GitHub で利用可能になりました。

🏢 2015 年 11 月 18 日、Microsoft は Visual Studio Online を「Visual Studio Team Services (VSTS)」に改名することを発表しました。

📜 Python はバージョン 3.5 で 2015 年に async/await サポートを追加しました。

📜 TypeScript はバージョン 1.7 で 2015 年に async/await サポートを追加しました。

🏢 Cloud Native Computing Foundation (CNCF) は Linux Foundation プロジェクトであり、2015 年に設立され、コンテナ技術を推進し、業界がその進化に沿って整合する為に支援することを目的としています。

🌐 2015 年に発行された HTTP/2 は、HTTP のセマンティクスのより効率的な「ワイヤー上の」表現を提供します。

## 2016

☁️ 2016 年 2 月、Helm パッケージマネージャー for Kubernetes がリリースされました。

☁️ Google Cloud Functions はサーバーレス実行環境であり、2016 年 2 月にアルファ版で利用可能になりました。

☁️ Azure IoT Hub は 2016 年 2 月 4 日に一般提供に達し、IoT アプリケーションとデバイス間の双方向通信の為の中央メッセージハブを提供します。

☁️ Kubernetes 1.2 は 2016 年 3 月にリリースされ、Deployments と最大 1,000 ノードのクラスタサポートを導入しました。

🖥️ Visual Studio Code はパブリックプレビューステージから卒業し、2016 年 4 月 14 日にバージョン 1.0 に達しました。その時点で、1,000 を超える拡張機能と 50 万人のアクティブユーザーがありました。

🔐 General Data Protection Regulation は GDPR、または French RGPD と略記され、欧州連合 (EU) および欧州経済領域 (EEA) の情報プライバシーに関する欧州連合規制です。
欧州議会および欧州連合理事会は 2016 年 4 月 14 日に GDPR を採択し、2018 年 5 月 25 日に発効する予定です。

☁️ 2016 年 4 月 20 日、Jenkins バージョン 2 がパイプラインプラグインをデフォルトで有効にしてリリースされました。このプラグインにより、Apache Groovy に基づくドメイン固有言語を使用してビルド命令を作成できます。

⚙️ Snap は Linux カーネルと systemd init システムを使用するオペレーティングシステム用に Canonical により開発されたソフトウェアパッケージングおよび展開システムです。
最初の snapd 2.12 は Ubuntu 16.04 (xenial-updates) および Fedora 24 にリリースされました。

🔐 Web Authentication (WebAuthn) は World Wide Web Consortium (W3C) により発行された Web 標準です。WebAuthn は FIDO Alliance の指導の下、FIDO2 Project のコア成分です。
最初に発行: 2016 年 5 月 31 日

🏢 2016 年 5 月、Cloud Native Computing Foundation は Prometheus をその 2 番目のインキュベートプロジェクトとして受け入れました。Kubernetes の後です。

🌐 ASP.NET Core は無料のオープンソース Web フレームワークであり、ASP.NET の後継であり、Microsoft により開発され、2016 年 6 月 7 日に初期リリースされました。

📜 Yarn は 2016 年に Facebook により Node.js JavaScript ランタイム環境用に開発されたソフトウェアパッケージングシステムであり、2016 年 6 月 18 日に初期リリースされました。

📜 \.NET Core 1.0 は 2016 年 6 月 27 日にリリースされました。これは Microsoft Visual Studio 2015 Update 3 とともにリリースされ、\.NET Core 開発を可能にします。

🖥️ Language Server Protocol (LSP) はオープンな JSON-RPC ベースのプロトコルであり、ソースコードエディタまたは統合開発環境 (IDE) とプログラミング言語固有の機能を提供するサーバー間で使用される為のものであり、元々 Microsoft Visual Studio Code 用に開発され、現在はオープン標準です。2016 年 6 月 27 日、Microsoft は Red Hat および Codenvy との協力を発表し、プロトコルの仕様を標準化します。

🖥️ VS Code Go 拡張は Go プログラミング言語の為の豊富な言語サポートを提供し、2016 年 7 月に初期リリースされました。

☁️ Prometheus 1.0 は 2016 年 7 月にリリースされました。その後のバージョンは 2016 年および 2017 年を通じてリリースされ、2017 年 11 月の Prometheus 2.0 につながりました。

⚙️ Windows Subsystem for Linux (WSL) は Linux バイナリ実行ファイル (ELF 形式) を Windows でネイティブに実行する為の互換性レイヤーです。
WSL ベータは Windows 10 バージョン 1607 (Anniversary Update) で 2016 年 8 月 2 日に導入されました。Ubuntu のみがサポートされていました (Bash をデフォルトシェルとして)。

🖥️ 2016 年 8 月 18 日、Microsoft は PowerShell がオープンソース化される (MIT ライセンスの下で) およびクロスプラットフォーム化されることを発表し、Linux および macOS 用のアルファバージョンがリリースされました。

⚙️ macOS 10.12 Sierra は 2016 年 9 月 20 日にリリースされました。Apple はオペレーティングシステムを「OS X」から「macOS」に改名して、その他のオペレーティングシステム (iOS、watchOS、tvOS) のブランディングと整合させました。また、Mac で初めて Siri を導入しました。

☁️ Cilium 0.1 はクラウドネイティブプロジェクトであり、eBPF を活用してコンテナ化されたワークロード用に高性能ネットワーキング、セキュリティ、および可観測性を提供し、2016 年 9 月にリリースされました。

🌐 Angular は TypeScript ベースの無料のオープンソース Web アプリケーションフレームワークであり、Google の Angular Team および個人や企業のコミュニティにより主導されています。Angular 2.0 は 2014 年 10 月 22 ~ 23 日の ng-Europe カンファレンスで発表され、2.0 バージョンの劇的な変更は開発者の間でかなりの議論を引き起こしました。バージョン 2.0 は 2016 年 9 月 14 日にリリースされました。

🧠 PyTorch は Torch ライブラリに基づく機械学習フレームワークであり、コンピュータビジョンおよび自然言語処理などのアプリケーション用であり、元々 Meta AI により開発され、現在は Linux Foundation の傘下にあります。初期リリースは 2016 年 9 月です。

🏢 Google Apps は 2016 年 9 月 29 日に正式に G Suite にブランド変更され、クラウドベースの生産性ツールの協調的で知的な性質を強調しました。

🌐 Next.js はオープンソース Web 開発フレームワークであり、Vercel により作成され、サーバーサイドレンダリング、および静的 Web サイト生成を備えた React ベースの Web アプリケーションを可能にし、2016 年 10 月 25 日に初期リリースされました。

🌐 Nuxt.js は無料のオープンソース JavaScript ライブラリであり、Vue.js、Node.js、Webpack、および Babel.js に基づいており、Next.js に触発され、React.js に基づいた同様の目的のフレームワークです。初期リリースは 2016 年 10 月 26 日です。

🌐 HTML 5.1 は 2016 年 11 月に W3C Recommendation として発行され、レスポンシブ画像用の `<picture>` 要素、`<details>` および `<summary>` 要素、および `inputmode` 属性などの機能を追加しました。

🏢 2016 年 11 月 15 日、Microsoft は Azure Functions の一般提供を発表しました。

🏢 Hugging Face, Inc. は機械学習を使用してアプリケーション構築の為の計算ツールを開発するニューヨーク市に本拠を置くアメリカの企業です。同社は 2016 年に設立され、元々十代向けのチャットボットアプリを開発した企業としてのものでした。Hugging Face Hub は中央集約された Web サービスをホストする為のプラットフォームです。

## 2017

📜 ISO は 2017 年に PDF 2.0 を ISO 32000-2 として発行しました。これは Adobe 独自拡張なしに ISO によって完全に開発された PDF 仕様の最初のバージョンであり、改善されたデジタル署名、関連ファイル、および地理空間データへのより良いサポートなどの新機能を導入しました。

🌐 k6 は 2017 年 2 月に Load Impact によりオープンソース化されたモダンな負荷テストツールであり、Go で書かれた JavaScript スクリプティング により設計され、開発者ワークフローと CI/CD インテグレーション の為のものです。Load Impact は後に 2020 年に k6 にブランド変更され、2021 年に Grafana Labs に買収されました。

🌐 WebAssembly (時々は Wasm と略記される) はポータブルバイナリコード形式および実行可能プログラムの為の対応するテキスト形式を定義します。
2015 年に発表され、2017 年 3 月に最初にリリースされました。

🏢 Google は 2017 年 3 月にデータサイエンティストおよび機械学習実務者の最大のコミュニティである Kaggle を買収しました。

🌐 WebAssembly System Interface (WASI) は Mozilla により設計されたシンプルなインターフェース (ABI および API) であり、あらゆるプラットフォームにポータブルになることを目的としています。これはファイル I/O などの POSIX 様の機能を提供し、能力ベースのセキュリティにより制約されています。

🧠 DVC は無料でオープンソース、プラットフォームに依存しないデータ、機械学習モデル、および実験の為のバージョンシステムであり、ML モデルの為に共有可能にし、実験を再現可能にし、モデル、データ、および パイプラインのバージョンを追跡する為に設計されており、2017 年 5 月 4 日に初期リリースされました。

☁️ Google Cloud IoT Core は 2017 年 5 月にベータ版で起動され、IoT デバイスの接続および管理の為のものです。

🐛 WannaCry ランサムウェア攻撃は 2017 年 5 月の世界規模のサイバー攻撃であり、WannaCry ランサムウェア cryptoworm により、Microsoft Windows オペレーティングシステムを実行しているコンピュータを標的にし、データを暗号化し、Bitcoin 暗号通貨でのランサム支払いを要求しました。

📊 Azure Cosmos DB はグローバルに分散されたマルチモデルデータベースサービスであり、2017 年 5 月 10 日に一般提供に達しました。

☁️ Kubernetes 1.6 は 2017 年 3 月にリリースされ、ロールベースアクセス制御 (RBAC) を導入し、etcd v3 をデフォルトストレージバックエンドとしました。

☁️ Kubernetes 1.7 は 2017 年 6 月にリリースされ、カスタムリソース定義 (CRD) をサードパーティリソース (TPR) の置換として導入しました。

🏢 「Microsoft 365」ブランドは 2017 年 7 月 10 日に Microsoft Inspire でエンタープライズサブスクリプション製品として初めて導入され、Office 365 と Windows 10 Enterprise ライセンスおよび Enterprise Mobility + Security (EMS) をバンドルしました。

🌐 2017 年 7 月、Adobe は 2020 年末までに Flash Player 全体の廃止の意思を発表しました。セキュリティ更新を含む です。これにより、すべての主要なブラウザーとプラットフォームが削除計画を最終化するよう促しました。

☁️ OCI v1.0 は 2017 年 7 月 19 日に正式にリリースされ、Runtime および Image Specifications の安定化をマークしました。runc 1.0 は OCI Runtime Specification の参照実装であり、同じ日にリリースされました。

📊 Apache Iceberg はオープンソース高性能形式であり、巨大な分析テーブル用であり、Spark、Trino、Flink、Presto、Hive、Impala、StarRocks、Doris、および Pig などのエンジンが同じテーブルで同時に安全に操作できるよう SQL テーブルを大規模データに使用できるようにし、2017 年 8 月 10 日に初期リリースされました。

📜 2017 年 9 月 12 日、Oracle Corporation は Java EE を Eclipse Foundation に提出すると発表しました。Oracle が「Java」の商標を保持していたため、後に Jakarta EE へと改称されました。

🌐 Java 9 は 2017 年 9 月 21 日にリリースされ、公式に Java Applet API (`java.applet` パッケージ) を廃止し、Java プラットフォーム内のアプレットサポートの正式な終了を示しました。

🌐 Cypress は 2017 年 5 月 4 日にオープンソース化されることを発表し、2017 年 10 月 10 日にパブリックベータに入りました。これは単一モノレポに再構成され、npm 経由でインストール可能な Node.js モジュールとして利用可能になり、3 年間のプライベート開発を終了しました。

⚙️ WSL は Windows 10 バージョン 1709 (Fall Creators Update) でベータではなくなり、2017 年 10 月 17 日にリリースされました。複数の Linux ディストリビューションをインストール可能であり、Windows Store でインストール用に利用可能でした。

⚙️ Ubuntu 17.10 (Artful Aardvark) は 2017 年 10 月 19 日にリリースされ、GNOME Shell インターフェースを使用し、X11 ウィンドウシステムを Wayland ディスプレイサーバーに置き換える最初のリリースをマークしました。このデフォルトは、Unity をデフォルトデスクトップとして使用してから約 6.5 年後に発生しました。

🏢 2017 年 10 月末、Microsoft は AKS (Azure Container Service) のプレビューを発表しました。これは Azure 内のマネージド Kubernetes サービスです。

☁️ Google Kubernetes Engine (GKE) は 2017 年 11 月に CNCF により認定され、より広いエコシステムとの互換性を確保しました。

⚙️ 2017 年 11 月 6 日、Amazon は Nitro と呼ばれる KVM ハイパーバイザーの周りのカスタムアーキテクチャに基づいた C5 ファミリーの新しいインスタンスを発表しました。

🧠 Amazon SageMaker はクラウド機械学習プラットフォームであり、開発者が機械学習 (ML) モデルをクラウド内で作成、トレーニング、およびデプロイできるようにし、開発者が ML モデルを組み込みシステムおよびエッジデバイスにデプロイできるようにします。これは 2017 年 11 月 29 日に起動されました。

🏢 Databricks は University of California、Berkeley の AMPLab プロジェクトから成長し、Scala 上に構築されたオープンソース分散コンピューティングフレームワークである Apache Spark の製造に関与していました。
2017 年 11 月、同社は Azure Databricks インテグレーション経由で Microsoft Azure 上のファースト パーティサービスとして発表されました。

🌐 HTML 5.2 は 2017 年 12 月に W3C Recommendation として発行され、`<dialog>` 要素および Payment Request API サポートを追加しながら、廃止された機能を削除しました。

📜 Javascript は 2017 年の ECMAScript 2017 JavaScript 版の一部として async/await のサポートを追加しました。

🧠 Transformer はディープラーニングモデルであり、自己注意機構を採用し、入力データの各部分の重要度に対して異なる重みを付けます。これは主に自然言語処理 (NLP) およびコンピュータビジョン (CV) の分野で使用されます。
Transformers は 2017 年に Google Brain のチームにより導入され、LSTM などの RNN モデルを置き換え、NLP 問題の為のモデル選択がますます です。

☁️ Kata Containers はオープンソースプロジェクトであり、コンテナのように見え、動作する軽量仮想マシンを提供し、2017 年 12 月 5 日に発表されました。これは Intel Clear Containers および Hyper.sh runV の統合として形成されました。

🌐 Uvicorn は Python 用の高速 ASGI (Asynchronous Server Gateway Interface) サーバー実装であり、Tom Christie により作成され、2017 年後半に最初にリリースされました。

## 2018

🖥️ PowerShell Core 6.0 は 2018 年 1 月 10 日に一般利用可能性に達しました。\.NET Core 上に構築され、Windows、macOS、および Linux でのクロスプラットフォーム使用の為に設計された最初のバージョンでした。

☁️ 2018 年 1 月 15 日、AWS は Go を AWS Lambda の為のサポート言語として発表しました。

🐛 Meltdown は 2 つの元のトランジェント実行 CPU 脆弱性の 1 つです (もう 1 つは Spectre です)。Meltdown は Intel x86 マイクロプロセッサー、IBM POWER プロセッサー、およびいくつかの ARM ベースのマイクロプロセッサーに影響します。これにより、悪意のあるプロセスは、許可されていない場合でも、すべてのメモリーを読み取ることができます。
発見日: 2018 年 1 月

🐛 Spectre は 2 つの元のトランジェント実行 CPU 脆弱性の 1 つ (もう 1 つは Meltdown です) を指し、マイクロアーキテクチャータイミング サイドチャネル攻撃を含みます。これらは分岐予測および他の形式の推測を実行するモダンなマイクロプロセッサーに影響します。
発見日: 2018 年 1 月

🧠 Generative Pre-trained Transformer 2 (GPT-2) は 2019 年 2 月に OpenAI により作成されたオープンソース人工知能です。
